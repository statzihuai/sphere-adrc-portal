# SPHERE ADRC Portal

A public-facing research portal for the Stanford Alzheimer's Disease Research Center (ADRC) synthetic dataset, generated by the **SPHERE** algorithm, a proprietary privacy-preserving synthetic data generation method developed at Stanford.

**Live portal:** https://web.stanford.edu/~zihuai/adrc-sphere/  
**PI:** Zihuai He, PhD — zihuai@stanford.edu · Stanford University

---

## What this is

The portal lets researchers worldwide explore a privacy-preserving synthetic twin of the Stanford ADRC cohort — 644 participants, 9 data modalities, 206,000+ features — with no IRB amendment or data-use request required. The original patient data never moves; only the SPHERE synthetic copy is exposed.

The portal includes a **SPHERE AI Agent** tab: an embedded AI analyst (powered by Claude) that lets users query and analyze the data in natural language, run Python code locally in the browser, and generate figures and reports.

## Getting started

### Requirements

- Python 3.10+
- Git LFS (for the large sphere/ data files)

### Clone

```bash
# Install Git LFS first
brew install git-lfs       # macOS
# or: sudo apt install git-lfs  (Linux)

git lfs install
git clone https://github.com/statzihuai/sphere-adrc-portal
cd sphere-adrc-portal
```

Git LFS automatically downloads all `portal/data/sphere/*.csv` files on clone (~1.2 GB total). The largest file is `scrna.csv` (1.1 GB).

### Run the portal locally

No build step needed. Open directly in browser:

```bash
open portal/index.html
# or just double-click portal/index.html in Finder
```

The portal loads data from relative paths — no local server required for the static frontend.

To use the **SPHERE AI Agent**, you need a Claude API key from [console.anthropic.com](https://console.anthropic.com). Paste it into the Connect screen. Your key is stored in `sessionStorage` only and never sent to Stanford servers.

### Rebuild the compiled data files

If you modify anything in `portal/data/sphere/` or `portal/data/dd/`:

```bash
cd portal/
python3 scripts/inject_data.py   # → rewrites data/adrc_data.js
python3 scripts/inject_dd.py     # → rewrites data/adrc_dd.js
```

## Architecture overview

```
Browser
├── Static portal (index.html + data JS files)
├── Pyodide (WebAssembly Python) — all data analysis runs locally
│     ├── Full SPHERE×2 data loaded into pandas DataFrames
│     ├── Matplotlib figures rendered in browser
│     └── Data dictionary (151K entries) in Python memory
└── SPHERE AI Agent
      ├── Claude API called directly from browser (BYOK)
      └── All computation stays on user's machine — data never leaves browser
```

**Key design principle:** Python execution and data analysis are entirely client-side. The user's browser downloads the data and runs all computations locally via Pyodide (WebAssembly). No analysis results are sent to any server.

## Data

All data in `portal/data/sphere/` is **SPHERE×2 synthetic data** — two rounds of SPHERE rotation applied to the real ADRC dataset. It preserves the statistical structure of the original cohort without exposing any real patient records.

| Modality | Participants | Full columns | In `adrc_data.js` |
|---|---|---|---|
| Demographics | 644 | 30 | 5 core columns |
| Cognitive scores | 634 | 74 | All |
| Blood biomarkers | 618 | 3 | All |
| Imaging (amyloid + tau PET) | 276 | ~577 | All |
| Whole genome sequencing | 396 | 167 SNPs | All |
| PBMC scRNAseq | 310 | ~193,000 | 21 marker genes |
| CSF proteomics | 142 | 5,284 aptamers | 5 curated |
| Plasma proteomics | 422 | 6,900 aptamers | 5 curated |

`adrc_data.js` contains a browser-feasible subset. The full-width CSVs in `portal/data/sphere/` are the complete SPHERE×2 output and are used by `inject_data.py` to rebuild `adrc_data.js`.

The real patient data is **not included** and never will be. It remains at Stanford.

## What needs to be built

This repo contains the current working portal. The next development phase adds:

1. **Backend API** — auth, credit system, Anthropic proxy
2. **On-demand full data loading** — fetch full CSVs from backend into Pyodide on demand (replacing the pre-compiled `adrc_data.js` subset)
3. **Shared prompt caching** — centralized Anthropic key so all users share a warm cache; users billed at retail, SPHERE pays discounted cache rates (~40% AI margin)
4. **Stripe payments** — $10 trial credits, $29/month subscription, credit packs

See `ENGINEER_BRIEF.md` for the full technical specification.
